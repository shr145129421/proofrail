# Sponsor Angles

## Open Track
Proofrail is a general trust layer for autonomous agents. It combines bounded execution, privacy-aware reasoning, and verifiable receipts in one system.

## Protocol Labs — Agents With Receipts / Let the Agent Cook
- Structured agent manifest: `agent.json`
- Structured execution log: `agent_log.json`
- End-to-end loop documented in artifacts and repo history
- Safety guardrails demonstrated through blocked-path execution

## Venice — Private Agents, Trusted Actions
- Core framing is private-by-default execution
- Output-focused trust model: reveal receipts and outcomes, not more context than necessary
- Explicit focus on preserving sensitive input context while still enabling action

## MetaMask — Delegations
- Core authority model is scoped and delegation-oriented
- The product thesis is that agents should act through bounded authority, not unlimited keys or unlimited permissions

## Base — Agent Services on Base
- Proofrail can be exposed as a reusable service to other agents and humans
- Verification, constrained execution, and receipt generation can all be packaged as a service layer

## Locus
- Strong conceptual fit around spend controls and agent-native accounting
- Current implementation emphasizes payment policy and budget bounds, but not yet deep Locus-native integration

## OpenServ
- Strong conceptual fit around coordination and service infrastructure
- Current implementation is more infrastructure-shaped than OpenServ-native

## Self Protocol
- Identity is important to the architecture and judging artifacts
- Current implementation is still pre-integration

## Filecoin
- Receipts and logs could map well to durable storage
- Current implementation is not yet storage-backed in a way that makes Filecoin essential

## Honest read
The strongest current sponsor-aligned angles are:
- Protocol Labs
- Venice
- MetaMask
- Open Track

The weaker current angles are the tracks that expect hard third-party integration rather than architectural fit alone.
